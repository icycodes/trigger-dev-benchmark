## How It Works

1. **`npm run run-pipeline`** — Triggers `resilient-pipeline-${trial_id}` with `{ input: "initial data" }` and prints `Run ID: <run_id>`.

2. Inside the parent task:
   - Calls `unreliableTask.triggerAndWait({ input, shouldFail: true })` — catches the expected error.
   - Creates a waitpoint token via `wait.createToken()`.
   - Writes the token ID to `waitpoint_token.txt`.
   - Pauses execution with `wait.forToken<{ correctedInput: string }>()`.

3. **`npm run complete-waitpoint`** — Reads the token from `waitpoint_token.txt` and calls `wait.completeToken(tokenId, { correctedInput: "fixed data" })`.

4. Execution resumes in the parent task:
   - Calls `unreliableTask.triggerAndWait({ input: correctedInput, shouldFail: false })`.
   - Returns `{ result: "Processed: fixed data" }`.
