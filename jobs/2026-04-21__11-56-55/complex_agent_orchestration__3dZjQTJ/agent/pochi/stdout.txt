### `researchAgent-${trial_id}` — Orchestration Loop

```
Entry (query, depth=1)
    │
    ├─ depth > 2? ──YES──► return "Max depth reached"
    │
    ▼
triggerAndWait(searchTask, { query })
    │
    ├─ result contains "insufficient"?
    │       YES ──► triggerAndWait(researchAgent, { query: "[refined] …", depth+1 })
    │                  └─ return recursive output
    │
    └─ NO ──► return { result, depth }
```

Each invocation:
1. Checks the depth guard (`depth > 2` → early exit).
2. Calls `triggerAndWait` to block until `searchTask` completes.
3. Inspects the output — if insufficient, constructs a `[refined]` query and recursively calls itself with `depth + 1` via `triggerAndWait`.
4. On a sufficient result, returns immediately.

All error paths from `triggerAndWait` (`.ok === false`) throw with a descriptive message.
