### `trigger/ai.ts` — what it does

**Streams defined (module-level, shared):**
| Stream | Kind | ID |
|--------|------|----|
| `aiStream` | Output (`streams.define<string>`) | `aiStream-cancellable_ai_stream__kcWqQS4` |
| `cancelStream` | Input (`streams.input<{ reason? }>`) | `cancelStream-cancellable_ai_stream__kcWqQS4` |

**Task `aiTask-cancellable_ai_stream__kcWqQS4`:**
1. **Cancellation listener** — `cancelStream.on()` registers a handler that sets `cancelled = true` and captures the reason the moment any message arrives on the input stream.
2. **Generation loop** — iterates 10 times; each iteration:
   - Checks `cancelled` flag first — breaks immediately if set.
   - Appends a text chunk to `aiStream` via `aiStream.append()`.
   - Waits 1 second via `wait.for({ seconds: 1 })` before the next chunk.
3. **Cleanup** — `subscription.off()` is called in a `finally` block to unsubscribe the cancel listener.
4. **Return** — returns `{ status: "cancelled", reason }` or `{ status: "completed" }` accordingly.

**To cancel a running task from outside:**
```ts
await cancelStream.send(runId, { reason: "User requested stop" });
```
