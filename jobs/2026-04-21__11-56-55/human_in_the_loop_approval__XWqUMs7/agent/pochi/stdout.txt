## Task: `trigger/approval.ts`

The task ID is `approvalTask-human_in_the_loop_approval__XWqUMs7` (using the `trial_id` read from `/logs/trial_id`).

**Workflow steps:**
1. Logs `"Requesting approval..."`
2. Creates a wait token via `wait.createToken({ timeout: "1h" })` — expires after 1 hour if not resolved
3. Logs the token ID (so an external caller can use it to resolve the token)
4. Suspends execution via `wait.forToken(token.id)` — the task pauses here until the token is completed
5. Checks `result.ok && result.output?.approved === true`:
   - **Approved** → logs `"Request approved!"`
   - **Rejected/timeout** → logs `"Request rejected."`

## How the approval flow works

An external system completes the token (approves or rejects) by calling the Trigger.dev API:

```bash
# Approve
curl -X POST https://api.trigger.dev/api/v1/waittoken/<token-id>/complete \
  -H "Authorization: Bearer <TRIGGER_SECRET_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"output": {"approved": true}}'

# Reject
curl -X POST https://api.trigger.dev/api/v1/waittoken/<token-id>/complete \
  -H "Authorization: Bearer <TRIGGER_SECRET_KEY>" \
  -H "Content-Type: application/json" \
  -d '{"output": {"approved": false}}'
```

The task then resumes and logs the appropriate result.
