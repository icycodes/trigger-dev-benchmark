## Key Files

### `trigger/progress.ts`
Defines the task `progress-task-001` (where `001` is the `trial_id`):

- **Task ID**: `` `progress-task-${trial_id}` `` — parameterised so it can be changed by modifying the `trial_id` constant or passing `trialId` in the payload.
- **5-step loop**: Iterates from step 1 to `totalSteps` (default: 5).
- **Real-time metadata updates** on every iteration using the `metadata` API:
  ```ts
  await metadata.set("progress", currentProgress);  // 20, 40, 60, 80, 100
  await metadata.set("currentStep", step);
  await metadata.set("lastUpdate", new Date().toISOString());
  ```
- Initialises metadata with `progress: 0` and `status: "running"` before the loop, and finalises with `progress: 100` and `status: "completed"`.

## How to Run

1. **Add your API key** to a `.env` file:
   ```
   TRIGGER_SECRET_KEY=your_secret_key_here
   ```

2. **Start the dev worker**:
   ```bash
   cd /home/user/progress-app
   npm run dev
   ```

3. **Trigger a run** from the dashboard or via API and watch the `progress` metadata update in real-time (0% → 20% → 40% → 60% → 80% → 100%).

The `progress` metadata key is updated at each step, making it consumable via the Trigger.dev dashboard or the [Realtime API](https://trigger.dev/docs/realtime) for live progress streaming.
