### What was built (`trigger/ai-agent.ts`)

#### 1. `schemaTask` — `refine-text-${TRIAL_ID}`
- Accepts `{ text: string, iteration?: number }` (validated with Zod).
- Returns `{ refined, iteration, changes }`.
- **Mock path** (default, no key needed): three progressive vocabulary-strengthening passes.
- **Real LLM path** (opt-in): set `OPENAI_API_KEY` + install `@ai-sdk/openai` to call GPT-4o-mini via the Vercel AI SDK's `generateText`.

#### 2. `ai.tool` wrapper — `refineTextTool`
```ts
export const refineTextTool = ai.tool(refineTextTask);
```
`ai.tool` from `@trigger.dev/sdk/ai` converts the `schemaTask` into a Vercel AI SDK `Tool`. The SDK:
- uses the task's Zod schema as the tool's `inputSchema` (via `zodSchema()`).
- uses the task's `description` as the tool description.
- calls `task.triggerAndWait(input).unwrap()` inside `execute`.

This tool can be passed directly to any `generateText` / `streamText` call's `tools` map.

#### 3. `schemaTask` — `agent-loop-${TRIAL_ID}`
The parent agent drives **at least 2 refinement passes** via a loop:
```
for i in 1..maxIterations:
    result = tasks.triggerAndWait("refine-text-<id>", { text, iteration: i })
    if text unchanged AND i >= 2 → break early
    currentText = result.output.refined
return { original, final, totalPasses, history }
```
Every pass is fully durable — if the worker restarts, Trigger.dev resumes from the last completed subtask.
