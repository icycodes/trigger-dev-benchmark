## Key Implementations

**1. Queue Configuration** (`trigger/queue.ts`):
- Defined `criticalQueue` with `concurrencyLimit: 1`
- This ensures only one task can run at a time globally for this queue

**2. Task Configuration** (`trigger/tasks.ts`):
- Created `exclusiveTask` that uses the queue
- Implemented `concurrencyKey` function that returns `resource:${payload.resourceId}`
- This ensures only one run per resource ID can execute at a time

## Concurrency Control Levels

1. **Queue-level**: Limits total concurrent runs across all tasks using this queue to 1
2. **Resource-level**: Limits concurrent runs per specific resource ID via the concurrencyKey

## Note

The queue and task names use `${trial_id}` as a placeholder. You should replace the placeholder strings in `trigger/queue.ts` and `trigger/tasks.ts` with your actual trial ID or environment identifier.
