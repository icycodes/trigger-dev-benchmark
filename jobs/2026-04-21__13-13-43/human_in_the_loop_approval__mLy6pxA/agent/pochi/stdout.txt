## How the workflow operates

### Task: `approval-workflow` (`trigger/approval.ts`)

1. **Triggered** with an `ApprovalPayload`:
   ```ts
   { trial_id: string, requestedBy: string, description?: string }
   ```

2. **Suspends** the run by calling:
   ```ts
   const { data } = await wait.forToken<ManagerDecision>(
     `manager-approval-${trial_id}`
   );
   ```
   The token key is namespaced per `trial_id`, so concurrent trials each get their own independent approval token.

3. **Resumes** when a manager completes the token externally:
   ```ts
   // From any backend / dashboard / API call:
   await client.tokens.complete("<token>", {
     approved: true,
     comment: "LGTM",
     decidedBy: "alice",
     decidedAt: new Date().toISOString(),
   });
   ```

4. **Returns** the full decision payload:
   ```ts
   { trial_id, requestedBy, description, decision: ManagerDecision }
   ```
