## Implementation Details

**File: `trigger/approval.ts`**

The approval workflow task includes:
- **Task ID**: `approval-workflow-${trial_id}`
- **Token-based pause**: Uses `wait.forToken("manager-approval")` to halt workflow execution pending human approval
- **Data return**: Returns the approval data received from token completion

```typescript
trigger.task({
  id: "approval-workflow-${trial_id}",
  run: async (payload, { io, wait }) => {
    // Wait for human approval using a token
    const { data } = await wait.forToken("manager-approval");

    // Return the data received from the token completion
    return {
      approved: true,
      approvalData: data,
      timestamp: new Date().toISOString(),
    };
  },
});
```

## How It Works

1. The workflow executes and reaches the `wait.forToken("manager-approval")` checkpoint
2. Execution pauses waiting for the token to be completed
3. A manager can complete the token through the Trigger.dev dashboard or API with approval data
4. The workflow resumes and returns the approval data
