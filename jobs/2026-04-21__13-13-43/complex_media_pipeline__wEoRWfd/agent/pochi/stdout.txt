## Implemented Features

### ✅ Four Tasks Defined

1. **download-task**: Downloads media from URL to destination
2. **process-task**: Processes media with specified operations
3. **upload-task**: Uploads processed media to destination(s)
4. **media-pipeline-main**: Orchestrates the three-step pipeline

### ✅ Three Sequential Steps

1. Trigger `download-task` and wait for result
2. Trigger `process-task` with download result and wait
3. Trigger `upload-task` with process result and wait

### ✅ Robust Error Handling

- Try-catch block wraps the entire orchestration
- Logs errors and returns failure status
- Identifies which step failed (`download`, `process`, or `upload`)
- Detailed error messages included in response

### ✅ Parallel Execution Support

- Uses `batch.triggerByTaskAndWait` for parallel uploads to multiple destinations
- Automatically detects when multiple destinations are provided
- Combines results from parallel tasks
